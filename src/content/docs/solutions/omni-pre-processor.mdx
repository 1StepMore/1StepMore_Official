# Omni Pre-Processor

结构化提取，格式不失真。

## 产品概述

Omni Pre-Processor（OPP）是壹目贯维内容生产流水线的「第一公里」——它解决的是 AI 读取文档时的质量问题：乱码、格式丢失、结构错乱。不同于简单的格式转换，OPP 在提取过程中保留了文档的语义结构，让 AI 拿到的是「干净、可信赖」的原始内容。

## 核心痛点

**文档喂给 AI 后，输出质量总是不稳定。**

这是因为源文档本身没有被「预处理」——PDF 的多栏布局被压成一团，Word 的标题层级在纯文本中消失，PPT 的图表说明和页面内容混在一起。AI 看到的不是「结构化文档」，而是一堆字符流。

OPP 的做法是：提取时就把结构还给文档。

## 核心能力

**格式智能识别**

自动识别 PDF、Word（.docx）、PPT（.pptx）等主流办公格式，无需人工指定。每一页、每一段、每一个表格都有自己的位置信息。

**结构化输出**

提取结果不是纯文本，而是带语义的 JSON / Markdown：标题、段落、列表、表格、图表说明各自独立，层级关系完整保留。这让 AI 在理解文档时不再需要「猜测」上下文。

**格式零损失**

图表、公式、特殊符号的处理经过专项优化，不会出现乱码或截断。输出可以直接进入知识库或喂给大模型，无需人工清洗。

**多格式导出**

支持导出为 Markdown（适合知识库）、XLIFF（适合翻译流程）、JSON（适合二次开发）、纯文本（适合快速处理）。一次提取，多种格式复用。

**骨架元数据保留（v0.2.0 新增）**

DOCX/PPTX 提取时自动保存原始 ZIP 骨架结构。配合 manifest.json（包含源文件名、工具版本、字符统计、骨架路径），每次提取都有完整溯源记录，版本回滚和差异对比成为可能。

**Manifest 清单（v0.2.0 新增）**

每次处理自动生成 manifest.json，包含源文件哈希、生成时间、工具版本、输出格式、字符统计等信息。便于 CI/CD 集成和质量审计。

## 技术架构

OPP 以 MCP Server 的形式提供（Model Context Protocol），可以无缝接入 Claude、ChatGPT 等主流 AI 助手，也可以通过 REST API 直接调用。安全设计内置 PathValidator，所有文件操作必须在授权目录范围内，防止路径遍历攻击。

## 适用场景

**知识库建设**

企业积累了大量历史文档（产品手册、技术文档、内部培训材料），直接喂给 AI 建知识库效果差。OPP 先将文档结构化提取，知识库的召回率和准确率大幅提升。

**AI 内容生产管道**

内容生产的上游是素材消化。OPP 把 PDF 报告、竞品分析、客户案例等文档转化为结构化输入，让 AI 在生产环节的输出质量更稳定。

**翻译工作流前置处理**

多语言本地化之前，需要对原始文档做结构化处理。OPP 的 XLIFF 导出格式可以直接对接主流 CAT 工具，保留格式的同时完成翻译预处理。
