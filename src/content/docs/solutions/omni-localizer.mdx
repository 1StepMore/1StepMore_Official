## Omni Localizer

不只是翻译，是真正的本地化。

## 产品概述

Omni Localizer（OL）是壹目贯维内容生产流水线的「国际化出口」。它解决的不是简单的「文字翻译」，而是让内容真正进入目标语言市场的本地化问题——格式保留、文化适配、术语一致性，一次完成。

## 核心痛点

**翻译后的内容还需要大量人工调整。**

市场上大多数翻译工具的产出是「字面对应」：语法正确，但读起来不像目标市场的用户在说话。数字格式、日期格式、货币单位、标题长度、CTA 话术——这些细节决定了本地化内容是否真正有效。

OL 的做法是：翻译之前先理解语境，翻译之后保留格式，术语在整个项目中保持一致。

## 核心能力

**语境感知翻译**

不只是翻译文字，而是理解内容的使用场景。同样是「了解更多」，在 B2B SaaS 的落地页和娱乐类博客的语气完全不同。OL 基于内容的实际用途调整翻译策略，确保产出读起来像目标市场的原创内容。

**格式完整保留**

翻译后，标题层级、段落结构、列表格式、表格排版全部保留。不会出现在原文是三行标题、译文变成一段长文的情况。交付物可以直接上线，无需设计师重新排版。

**术语库管理**

在项目级别建立术语库，确保关键品牌词汇、产品名称、技术术语在所有页面保持一致译法。第一次翻译时定义，之后所有内容自动遵循，避免了多人协作时的术语混乱。

**多格式支持**

支持 Markdown、HTML、JSON、XLIFF 等常见内容格式的输入输出。可以作为独立工具使用，也可以通过 MCP 接口无缝嵌入现有内容生产管道。

**AI Agent 驱动**

OL 的核心是 AI Agent，不是一个简单的翻译 API。它能理解上下文、处理复杂的长文档、处理同一个词在不同场景下的不同译法——这是传统机器翻译做不到的。

**YAML 元数据头（v0.2.0 新增）**

Markdown 文件翻译完成后自动生成 YAML frontmatter 元数据块，包含源语言、目标语言、源文件名、工具版本等信息。方便追溯来源、接入 CI/CD 流程。

**XLIFF 头部注释（v0.2.0 新增）**

支持在 XLIFF 文件 Header 中注入译者注释（`<header><note from=OL>`），保留翻译过程元数据，兼容主流 CAT 工具。

## 技术架构

OL 以 MCP Server 的形式提供（Model Context Protocol），可以接入 Claude、ChatGPT 等主流 AI 助手，也可以通过 CLI 快速处理单个文件。支持 Docker 部署，满足企业对数据安全的合规要求。

## 适用场景

**产品国际化**

将中 文产品页面、帮助文档、营销文案本地化为英语、日语、韩语等多语言版本，直接上线。保留格式，术语一致，读起来像目标市场的原创内容。

**内容出海团队**

内容运营团队需要持续产出多语言社交媒体内容。OL 接管翻译环节，团队专注内容策划，翻译质量和速度同步提升。

**企业级多语言知识库**

跨国企业内部有大量多语言文档需要统一管理。OL 将不同语言的文档结构化入库，支持多语言检索和知识共享。
